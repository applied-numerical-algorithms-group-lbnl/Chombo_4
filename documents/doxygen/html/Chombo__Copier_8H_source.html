132 /// A strange but true thing to make copying from one boxlayoutdata to another fast
133 /**
134  class to handle the organization of data from a LevelData to a BoxLayoutData
135  defined over the same index space. The idea behind this object
136  is twofold:
137  A) there is a significant amount of information that can be
138  computed once and reused from one copy to the next for
139  a LevelData and BoxLayoutData that have unchanging BoxLayouts
140  underneath them. In these cases, LevelData::copyTo methods
141  has an optional argument. That way, in cases where the operation
142  is only performed a single time, we do not waste our time building
143  optimizing data, etc.
144  B) when we interface with KeLP, this class maps quite well
145  with the existing KeLP API (class MotionPlan).
146 
147  pains have been taken to not have this class be templated. Thus
148  it does not actually perform the copying itself (which would require
149  knowledge of the template class type). It provides an iterator
150  interface to MotionItems that specify the required work. LevelData
151  provides the copyTo methods.
152 */
