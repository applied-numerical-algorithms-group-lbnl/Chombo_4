120 /// A class to facilitate interaction with physical boundary conditions
121 /**
122 
123  ProblemDomain is a class which facilitates the application of physical
124  boundary conditions, both periodic and non-periodic. This class contains
125  much of the functionality of the Box class, since logically the
126  computational domain is generally a Box.
127 
128  Intersection with a ProblemDomain object will result in only removing
129  regions which are outside the physical domain in non-periodic directions.
130  Regions outside the logical computational domain in periodic directions will
131  be treated as ghost cells which can be filled with an exchange() function
132  or through suitable interpolation from a coarser domain.
133 
134  Since ProblemDomain will contain a Box, it is a dimension dependent class,
135  so SpaceDim must be defined as either 1, 2, or 3 when compiling.
136 
137  Note that this implementation of ProblemDomain is inherently
138  cell-centered.
139 
140 */
