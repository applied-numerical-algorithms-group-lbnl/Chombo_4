244 /// Data on a BoxLayout
245 /**
246 
247  A collection of Box-oriented objects. The arrangement
248  of Boxes is given by the underlying BoxLayout object.
249  BoxLayoutData attempt to prevent users from manipulating
250  the template class T to become out of sync with the
251  boxes in the BoxLayout. Caveat emptor.
252 
253  All access to the data in a BoxLayoutData is forced to be
254  data-parallel, by use of the DataIterator.
255 
256  class T must provide the following methods:
257  - <PRE> T() </PRE>
258  - <PRE> T(const Box& box, int comps) </PRE>
259  - <PRE> define(const Box& box, int comps) </PRE>
260  - <PRE> void copy(const Box& Rfrom, const Interval& Cd, const Box& Rto, const T& source, const Interval Cs)
261  Cs and Cd must have same length, but can be displaced;
262  Rfrom and Rto must be the same size, but can be translated. </PRE>
263  - <PRE> static int preAllocatable()
264  returns 0 if the size(...) function is strictly a function of Box and
265  Interval, and does not depend on the current state of the T object.
266  return 1 if size(..) is symmetric, in that sender and receiver T object
267  can size their message buffers, but a static object cannot.
268  returns 2 if the object is truly dynamic. the message size is subject to
269  unique object data. </PRE>
270  - <PRE> int size(const Box& R, const Interval& comps) </PRE>
271  - <PRE> void linearOut(void* buf, const Box& R, const Interval& comps)
272  writes the T object into the byte stream buf, already allocated. </PRE>
273  - <PRE> void linearIn(void* buf, const Box& R, const Interval& comps)
274  reads in the T object from the byte stream buf. </PRE>
275 
276  In sequential codes, many of these functions will not be called, but in the
277  general parallel case, they all will be.
278 */
